What Phase 6 requires
1. ~/.wiki/config.toml schema
The global config lives at ~/.wiki/config.toml and holds a [[wikis]] array:


[[wikis]]
name = "work"
path = "/Users/me/work-wiki"
default = true

[[wikis]]
name = "research"
path = "/Users/me/research-wiki"
default = false
remote = "git@github.com:me/research-wiki.git"   # optional
WikiEntry struct: name: String, path: PathBuf, default: bool, remote: Option<String>.
WikiRegistry struct holds Vec<WikiEntry>.

2. registry.rs — implement WikiRegistry
WikiRegistry::load(config_path: &Path) -> Result<WikiRegistry> — parse the TOML above
WikiRegistry::resolve(name: Option<&str>) -> Result<&WikiConfig> — returns WikiConfig { root, name } for the matching entry:
None → return entry with default = true; error if none has default = true
Some(name) → find by name; error listing all available names if not found
Validation on load: error if two entries both have default = true; error if path does not exist (surface the wiki name in the error message)
3. wiki init --register flag
Add --register flag to the existing Init command in src/cli.rs. When set, append the newly created wiki to ~/.wiki/config.toml after init (create the file if absent). First wiki added becomes default = true automatically.

4. --wiki global flag threading
--wiki <name> already exists in src/cli.rs as pub wiki: Option<String> on Cli. Currently it is ignored everywhere. Thread it through:

Load WikiRegistry from ~/.wiki/config.toml (if it exists)
Call registry.resolve(cli.wiki.as_deref()) to obtain the WikiConfig
Fall back to WikiConfig { root: current_dir(), name: "wiki" } when the global config does not exist (single-wiki mode, unchanged behaviour)
Apply to: ingest, search, context, lint, list, contradict, graph, diff.

5. Cross-wiki search — search_all
Add to src/search.rs:


pub struct SearchResultWithWiki {
    pub wiki_name: String,
    // all fields from SearchResult: slug, title, snippet, score, page_type
}

pub fn search_all(
    registry: &WikiRegistry,
    query: &str,
    limit: usize,
) -> Result<Vec<SearchResultWithWiki>>
Fan out search(query, &entry.root, false) to every registered wiki, tag each result with wiki_name, merge and sort by descending score, truncate to limit.

wiki search --all "<term>" — calls search_all; results table gains a WIKI column.
